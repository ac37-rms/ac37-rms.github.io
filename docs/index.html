SensorSubscription

A subscription to a sensor.

max_data_rate is a 'best effort' and will only return as fast as is measured.

FieldTypeLabelDescription
sensor_id string

the ID of the sensor to receive data from

max_data_rate int32

optional. the maximum data rate, in Hertz (Hz), to receive data from this sensor.

SubscriptionRequest

A Subscription request.

Only one subscription is valid per client connection, any additional subscriptions will overwrite the previous.

If clients are requesting POST (non-live) data, use of stream_to/stream_from is required.

At least one of sensors and race_id is required, both can be specified.

FieldTypeLabelDescription
stream_from uint64

optional. the epoch in milliseconds from which to get POST data

stream_to uint64

optional. the epoch in milliseconds to get POST data up til (inclusive)

sensors SensorSubscription repeated

a list of SensorSubscriptions

race_id int32

optional. the ID of the race to subscribe to configuration updates to
